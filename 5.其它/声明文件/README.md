## 声明文件
```js
/*
  声明文件：
    -> 为JS代码提供类型声明
    背景：
      1.使用TS开发的库最终编译后就是JS，当使用这些库的时候就没有'类型检查'; 
      2.开发中使用某些库是JS编写，项目中使用也希望具备'类型检查'能力 => 书写声明文件便可以解决此类问题

    -> 声明文件的格式：以'.d.ts'结尾即可

    -> 声明文件的位置[执行tsc编译命令时，其会自行读取声明文件]
       1.放置在tsconfig.json配置的include包含的目录下即可
       2.放置在node_modules/@types目录下即可，@types是TS官方提供的包，已经包含了很多声明文件
       3.手动配置'读取目录'，tsconfig.json中配置typeRoots: ['./types1','./types2']
       4.放置在JS代码所在目录下，并且文件名也相同即可[实际并非强约束，不同名也OK] -> 推荐使用此方式，目前基于TS的工程项目都使用此方式

    -> 编写声明文件：
       1.自动生成：基于TS的工程项目，配置declaration:true即可，ts会自动生成相关的声明文件
       2.手动编写：基于JS的工程项目，全部重写为TS的成本很大，因此可手动编写声明文件，其它项目引入该项目时便具备'类型检查'能力

       -> 声明文件就是用于'类型声明'，并不是写ts代码，因此不要出现赋值、定义这类语句
*/
```
