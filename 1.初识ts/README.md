## 初识ts

> 安装 && 使用
```js
/*
  安装：
    npm install typescript -g
    tsc -v
       
  编译：
    1.命令行参数：tsc index.ts --参数;//tsc index.ts --watch
    2.配置文件：tsc;//默认读取配置文件tsconfig.json
    -> 实际TS有很多默认配置项，例如：执行环境基于浏览器、若代码没有使用模块化语句import/export,便认为该代码是全局执行、编译的目标代码是ES3，可以使用命令行参数或者配置文件的方式修改，但为了更好的移植性，开发中大多直接使用'配置文件'
    -> 优先级：自定义方式[命令行参数、配置文件] > 默认配置

  最终呈现形式：
    ts源文件 -> 编译[配置(默认参数、命令行参数、配置文件三者合并)] -> js目标文件;//配置文件最nice，移植性好

  补充：
    ts默认会检查项目工程根目录下所有文件，所以当不同文件中使用同名变量，其也会报错
    解决方案：
      1.尽量使用不同名变量
      2.使用立即执行函数实现模块化，防止变量冲突
*/
```
